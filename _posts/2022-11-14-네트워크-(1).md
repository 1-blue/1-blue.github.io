# IP 주소
`Host`에 대한 식별자<br />
`Network Id` + `Host Id` ( 32bit => 약 43억개 )

> `Host`란 인터넷에 연결된 컴퓨터

## 서브넷 마스크
`IP 주소`에서 `Network Id`의 길이를 표현하는 값<br />
비효율적인 클래스 방식을 파훼하기 위해 사용

# Port 번호
`Process`에 대한 식별자
( 16bit => 65535개 ( 0, 65535번은 사용하지 않음 ) )

# Switch
`LAN`에서 통신하기 위해 사용

+ `L2 Access Switch`: `endPoint`랑 직접 만나는 `Switch`
+ `L2 distribution Switch`: `L2 Access Switch`랑 연결된 `Switch`
+ `up-link`: 상위 스위치로 연결하는 라인

# MAC 주소
`NIC`에 대한 식별자, 하드웨어의 주소 ( 48bit )<br />
`LAN`에서 목적지를 찾기 위해서 사용

# Packet
네트워크에서 전달할 때 사용하는 데이터 집합체<br />
`header` + `payload`로 이뤄짐 ( `payload`는 상위 프로토콜에 맞는 `header` + `payload`로 이뤄짐 )

## 패킷 생성 과정
1. ( `application`계층(`L7`) ) `Stream` ---`segmentation`---> `Segment`
2. ( `transport`계층(`L4`) ) `Segment` ---`encapsulation`---> `Packet`
3. ( `Network`계층(`L3`) ) `Packet`---`encapsulation`---> `Frame`

## 계층 별 패킷을 부르는 명칭
`L4` => `Segment`라고 부르며, 최대 `MSS`(`maxium segment size`)만큼의 크기를 가짐 ( `MTU`에 의해 결정됨 )
`L3` => `Packet`라고 부르며, 최대 `MTU`(`maxium transport unit`)만큼의 크기를 가짐 ( 일반적으로 1500`byte` )
`L2` => `Frame`



========

+ `link-down`: 연결 끊어짐
